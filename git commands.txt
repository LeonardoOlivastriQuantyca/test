git init - Fai avviare git
git add - Crei una repo con i file che indichi (se indichi "." allora crei una repo con tutti i file di un progetto)
git diff - trovi la differenza tra una versione e un'altra. Se si aggiunge --staged farà vedere solo le modifiche ai file aggiunti al prossimo commit (i file staged, appunto)
git checkout - torni a una versione precedente
git switch - cambia il branch su cui si sta lavorando
git clone - copia tutta una repo in locale
git log - log dei commit storici

git config - configura alcuni parametri di Git, come utente o email

git remote add origin - linka la tua repo locale a una presente online, in modo da salvare le tue modifiche su cloud

git push -u "destinazione" - pushare il proprio codice sulla repo indicata sotto "destinazione"

git commit - salva le modifiche fatte. Se si aggiunge il parametro -a, non sarà necessario lanciare prima il comando "git add", e tutte i file modificati verranno committati (QUINDI OCCHIO)
